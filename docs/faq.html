FAQ

Common questions

Is this a commercial product?

No. The repository presents a transparent reference instrument intended for learning, validation, experimentation, and integration work.

Does it replace certified agronomic tools?

No. It is not positioned as a certified measurement device and should not be treated as the sole basis for agronomic or environmental decisions.

Why are there two AS7265x sensors?

One sensor measures incident light and one measures reflected light. That arrangement improves repeatability when ambient outdoor lighting changes.

What does the display show?

The intended interface shows a large NDVI value, battery percentage, and a low-light warning when the signal is too weak for a confident reading.

Which firmware should be treated as current?

The active ESP32 project is the PlatformIO layout in `firmware/src/`. The older `.ino` sketch is still present as a reference artifact.

Does the project include proprietary E-Nano pipelines?

No. Controlled calibration workflows, QA systems, and proprietary analysis layers are outside the scope of this open repository.

Build

Setup questions

  • Use the custom carrier PCB rather than assuming generic breadboard wiring.
  • Review the hardware PDFs and BOM before ordering parts.
  • Use the local patched AS7265X library included in `firmware/lib/`.

Measurements

Interpretation questions

  • NDVI is context-sensitive and not a diagnosis by itself.
  • Mixed soil, shadows, and directional lighting can distort readings.
  • Repeatability improves when both sensors are functioning correctly.
