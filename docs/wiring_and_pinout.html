Wiring and Pinout

The current firmware uses two I2C buses, one SPI display connection, a dedicated measure button, display backlight control, and battery sensing.

Dual I2C buses SPI display interface ESP32 Feather V2 target

Sensors

Why two I2C buses are used

The firmware treats the upward-facing and downward-facing AS7265x modules as separate buses. That avoids address conflicts and supports the dual-sensor incident-versus-reflected measurement approach.

  • UP sensor: ambient or incident illumination
  • DOWN sensor: reflected signal from the target surface

Display

Display integration

The interface is built around a 240x240 ST7789 screen. The active firmware controls the backlight separately and expects a no-reset configuration. 

TFT_CS  = A0
TFT_DC  = A1
TFT_RST = -1
TFT_BL  = A5

System notes

Additional interface details

  • The active BLE advertising name in firmware is `Manti-S-00001`.
  • The measure button and battery ADC live on analog-capable Feather pins.
  • The wiring should follow the carrier PCB rather than freehand prototyping assumptions.
  • Always cross-check this page against the board files in `hardware/fusion_eagle/`.
The pin mapping on this page reflects the current `firmware/src/main.cpp` that was copied into the repository. If firmware pin definitions change, this page should change with them.
